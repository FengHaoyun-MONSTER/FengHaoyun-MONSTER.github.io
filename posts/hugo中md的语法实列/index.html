说明#

  • draft：

  * true：草稿（本地可见，线上不发布）

  * false：正式发布

  • slug：URL 路径（强烈建议使用英文）

  • tags / categories：PaperMod 支持

二、标题规范#

#符号前面不能有任何东西,否则hugo不能正常渲染

# 一级标题（通常不用）

## 二级标题（正文主标题）

### 三级标题

#### 四级标题

注意事项#

  • ## 测试 和 ## 测试 效果相同（多余空格会被忽略）

  • 不要跳级（不要直接从 ## 跳到 ####）

三、段落与换行（非常重要）#

正确的段落写法#


这是第一段。



这是第二段。

效果：

这是第一段。

这是第二段。

强制换行（推荐方式）#


第一行␠␠

第二行

行尾 两个空格 + 回车

❌ 不推荐写法#


标题1 <br/>

原因：

  • HTML 标签在某些主题下可能被过滤

  • 可读性差

四、代码块（最常用）#

标准代码块（强烈推荐）#


```go

import "fmt"



func main() {

&nbsp;	fmt.Println("hello world")

}

```

规则（非常关键）#

  1. 三个反引号 必须单独一行

  2. 不要缩进代码块

  3. 语言标识写在开头（如 go、bash、json）
